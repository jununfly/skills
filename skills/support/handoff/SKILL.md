# Session Handoff

会话结束前生成交接文档，让下一个 Agent（或下一个会话的你）能无缝继续。

## 触发词

- "交接" / "handoff"
- "生成交接文档"
- "准备下一个会话"
- 用户明确说要结束当前会话

## 工作流程

### 1. 收集上下文

按优先级收集以下信息（已存在则引用路径，不复制内容）：

- [ ] 当前会话完成了什么（用户确认的目标）
- [ ] 关键决策和 ADR（如有，引用路径/URL）
- [ ] 未完成的任务和待办（task list 状态）
- [ ] 已知问题和阻塞（issues、failed tests）
- [ ] 相关产物路径（PRD、计划、diff、commit）
- [ ] 用户传递的下一个会话关注点（如有参数）

### 2. 生成交接文档

文件路径：`{用户指定目录}/handoff-{YYMMDD}.md`

**写入前先 Read 该文件** — 如果已存在，追加而非覆盖。

文档结构：

```markdown
# Handoff - {日期}

## 会话目标
> 本次会话要完成什么

## 已完成
- 已完成的事项（链接到产物，不粘贴内容）

## 关键决策
- 决策 + 理由（引用 ADR 路径，不重复内容）

## 待办 / 下一步
- [ ] 未完成的任务
- [ ] 建议下一个会话优先处理的事项

## 已知问题
- 阻塞项、failed tests、需要人工判断的问题

## 相关产物
- PRD: `{路径或 URL}`
- 计划: `{路径或 URL}`
- 最新 commit: `{hash 或 URL}`
- Diff: `{路径或 URL}`

## 建议下一个会话使用的技能
- `{skill-name}`: 为什么需要它

## 注意事项
- 不要踩的坑
- 环境要求（Rust 版本、node 版本等）
- 不要保存敏感信息（已检查）
```

### 3. 验证

- [ ] 没有粘贴 PRD / 计划 / ADR 的全文，只引用路径
- [ ] 没有 API key、密码、个人身份信息
- [ ] 下一个 Agent 读完后能知道从哪里继续

### 4. 建议下一个会话的技能

根据待办内容，从已安装 skill 列表中推荐 1-3 个最相关的技能，并说明原因。

## 参数处理

如果用户传递了参数（如 `"下一个会话 focus 在 Rust 迁移 Slice 7"`），将其写入"会话目标"和"待办/下一步"章节，并据此推荐技能。

## 禁忌

- 不复制已在其他产物中捕获的内容（PRD、计划、ADR、issue、commit、diff）→ 引用路径
- 不保存敏感信息（API keys、密码、PII）
- 不生成模糊的"待办" — 每条必须可执行
