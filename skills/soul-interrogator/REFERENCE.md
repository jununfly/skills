# Reference — Soul Interrogator

## 维度一：第一性原理拆解

逐节点推进，节点间不得跳跃。

### Node 1.1 — 识别核心假设

**问题**：这个想法建立在哪些底层假设之上？请列出你认为「不需要证明就成立」的前提。

**推荐解法**：用 "如果……就不成立" 句式反推。例如「做一款 AI 记账 App」，假设可能是 "用户愿意手动输入每一笔账" → 如果不成立，整个产品逻辑就塌了。

**共识确认**：用户列出的假设 + 你追问补充的假设，整理为清单。确认完整后进入 Node 1.2。

---

### Node 1.2 — 逐个挑战假设

**问题**：清单中的第 N 条假设——它一定成立吗？有什么反例或边界条件？

**推荐解法**：对每条假设追问三个子问题：
- 这个假设在过去 5 年有被推翻过的案例吗？
- 如果这个假设只在 80% 的情况下成立，剩下 20% 会怎样？
- 有没有一批用户/场景，这个假设对它们完全不成立？

**共识确认**：每条假设标注「稳固 / 有裂痕 / 已推翻」，确认后进入 Node 1.3。

---

### Node 1.3 — 重构路径

**问题**：把标注为「有裂痕」或「已推翻」的假设全部拿掉，从零开始，解决同一个问题还有什么方式？

**推荐解法**：强制发散，至少给出 3 条替代路径，不要求可行，只要求方向不同。例如「不用手动记账 → 银行 API 自动抓取 / 拍照 OCR / 语音输入」。

**共识确认**：挑选 1-2 条值得认真考虑的替代路径，或确认原路径仍最优。进入 Node 1.4。

---

### Node 1.4 — 维度一小结

**问题**：经过拆解→挑战→重构，你对这个想法的底层逻辑有什么新的认识？

**推荐解法**：用一句话总结：「我原本以为 ____，但现在意识到 ____。」

**共识确认**：用户陈述小结，复述确认。维度一结束。

---

## 维度二：红队攻击

角色切换：你是「最希望这个方案失败的人」。

### Node 2.1 — 最致命的单一攻击点

**问题**：只允许选一个角度摧毁这个方案，你会选什么？为什么最致命？

**推荐解法**：从四个方向扫描：
- 市场风险（没人需要 / 付费意愿为零）
- 执行风险（做不出来 / 做出来太慢）
- 竞争风险（巨头碾压 / 替代品更好）
- 外部风险（政策 / 技术变革 / 黑天鹅）

**共识确认**：确认最具杀伤力的 1 个攻击点，进入 Node 2.2。

---

### Node 2.2 — 连锁失败路径

**问题**：如果那个攻击点真的发生了，会如何引发连锁崩溃？

**推荐解法**：多米诺推演，至少 3 步：
> 攻击点触发 → A 失效 → B 无法支撑 → C 崩溃 → ……

**共识确认**：确认连锁链条合理完整，进入 Node 2.3。

---

### Node 2.3 — 防御方案

**问题**：面对攻击点和连锁链条，你能提前做什么？

**推荐解法**：三级防御——
- **预防**：让攻击点根本不发生（e.g. 先验证付费意愿再做产品）
- **缓冲**：攻击发生但不致命（e.g. 储备 12 个月 Runway）
- **逃生**：攻击后能体面退出（e.g. 保留回归职场的技能和人脉）

**共识确认**：为每级确定至少一个具体行动。进入 Node 2.4。

---

### Node 2.4 — 维度二小结

**问题**：经历红队攻击后，信心有什么变化？

**推荐解法**：量化表达：「攻击前信心 8/10 → 攻击后 X/10，主要因为 ____。」

**共识确认**：确认结论。进入综合审查报告。
