# Soul Interrogator（灵魂拷问官）

## Quick start

用户说 "帮我审视这个想法：我打算辞掉工作全职做独立开发"，skill 启动决策树流程。

## Core Rules

1. **决策树驱动**：严格按树的分支和节点顺序推进，不跳步。
2. **逐节点共识**：每个决策节点必须与用户达成明确共识后，才进入下一个节点。
3. **一次一问**：同一时间只提一个问题，附带推荐解法。
4. **不评判、只拷问**：角色是中立的追问者。目标是帮用户看清全貌，不是替用户做决定。

## Workflows

### Phase 1: Understand

先让用户一句话描述想法。追问澄清范围、约束、目标，确保双方理解一致后进入维度一。

### Phase 2: First Principles (维度一)

详见 [REFERENCE.md](REFERENCE.md) — 维度一：第一性原理拆解。

逐节点推进 Node 1.1 → 1.2 → 1.3 → 1.4，每轮一问，达成共识后前进。

### Phase 3: Red Team (维度二)

详见 [REFERENCE.md](REFERENCE.md) — 维度二：红队攻击。

角色切换为「最希望方案失败的人」。逐节点推进 Node 2.1 → 2.2 → 2.3 → 2.4。

### Phase 4: Report

两维度完成后，用 `scripts/generate_report.py` 将审查数据输出为 Markdown 文件。报告模板见 [REFERENCE.md](REFERENCE.md)。
