# 关于个人知识库的构建
## 个人知识体系的构建回顾
有很多文章讨论，不赘述。
我之前稳定在基于OneNote的一套个人实践上。为了充分照顾基于元认知的explore、exploit，做了个基本的设计：

| 笔记名前缀   | 归类                         | 内涵                 |
| ------- | ---------------------------- | ------------------ |
| 00   | ZERO      | 全局无状态信息 |
| A00 | 外脑 | 客观时间流；<br>当前的context；<br>一段特定时空的外视；<br>认知、思想、观点上历史验证可信赖的客观总结；<br>理念、原则、信条上历史验证可信赖的前人经验；<br>具备科学的发展历程和知识体系背景的方法论；<br>可复用的持续迭代中<br>工具，如框架、设计、实践等；<br>一段特定时空的内视；<br>我的综合总结提炼，如PPT、文章等；    |
| B00     | 热知识库               | 重点关注或亲历的知识库；<br>带Topic分类和时间线的，经过整理的知识库；<br>如读书笔记、网摘、博客、社区文章等，应尽量带来源，定期做内容裁剪；           |
| C00      | 冷知识库               | 只是因为触及或需要了解，偶尔关注到的知识库；<br>带Topic分类和时间线的，经过整理的知识库；<br>如读书笔记、网摘、博客、社区文章等，应尽量带来源，定期做内容裁剪；           |
| D00    | 碎片化        | 未分类的、碎片化的内容。<br>应尽快整理、沉淀或丢弃。       |

具体每篇文档，会在很多细节上做自我提示的设计，exp：<br>
都会尽量用它“诞生”的 YYYYMMDD 作为前缀，比如某本书的读书笔记会有它出版日期，这样方便快速知道内容的时效性。

之前这套运行得不错，不论多端同步、富内容组织、随使用热度串联信息点...，对我来说，学无止境、永远无知，所以对冷掉的信息点也没有焦虑感。
痛点也有，知识团/簇 过时时、有更新迭代时、甚至有定义/概念级改变时，比较难与时俱进去维护，又很难花足够多空闲时间去重新整理，复用效率会变差。
AI阶段开启后，方便AI阅读使用的.md文档+llm可维护+知识图谱化 变得必要。
不知道其他人有没有这种“小火”，就想重新设计下。

## 知识管理方法论回顾
### PARA
项目（Projects）、领域（Areas）、资源（Resources）、档案（Archives）。根据日常活动的使用热度，从高频到低频。
### CODE
收集（Collect）、整理（Organize）、提炼（Distill）、表达（Express）。根据信息的科学处理过程。
### DIKW
数据（Data）、信息（Information）、知识（Knowlege）、智慧（Wisdom）。根据一段字节流，对个人的意义、价值权重分层。
### ITKG
Intention Tower and Knowledge Graph。意义之塔 & 知识图谱。用存分离的一种思路，意义之塔匹配使用者的思维结构、认知习惯，知识图谱按ontology分布。
### Cards
卡片化，有一本书《卡片笔记写作法：如何实现从阅读到写作》；有一个app叫浮墨，fomo。
### Markdown & Obsidian & LLM
单文件过大、文件数过多时，哪哪都是问题。
### 总之
方法和工具试过很多，总而言之是个够用就行、忍不了就攒新版本的事。
人和Agent挺像的，瞬时context的大小有限，自身认知、注意力等在不断变化，对工具和方法过于“洁癖”容易主次颠倒 也不好。

### Obsidian vaults -> topics
```
Obsidian vaults/
├── life/              # 生活
├── work/              # 工作
├── tech/              # 技术
└── AI-Coding/         # AI-Coding
```

### vault -> topic
```
vault/
├── raw/              # 原始资料（不可变，LLM 只读不写）
│   └── assets/       # 图片等附件
└── wiki/             # LLM 生成的 wiki 页面
    ├── entities/     # 实体页面（人物、地点、组织）
    ├── concepts/     # 概念页面（想法、理论、方法论）
    ├── sources/      # 资料摘要
    ├── synthesis/    # 综合分析
    ├── index.md      # 索引
    └── log.md        # 操作日志
```
