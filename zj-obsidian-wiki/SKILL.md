# zj-obsidian-wiki - Obsidian 知识库管理

为 Obsidian 增、删、改、查个人 wiki 知识库。基于 llm-wiki.md 理念，让 LLM 维护一个持续进化的 wiki。

## 核心理念

不是每次都从零开始 RAG，而是**维护一个持续进化的 wiki**：

- 原始资料放在 `raw/`（不变）
- LLM 在 `wiki/` 中构建结构化知识
- 知识被编译一次，然后持续更新
- 交叉引用、矛盾标记、综合分析都已存在

## 何时使用

* 需要管理个人知识库、研究笔记、读书摘要时
* 有新资料需要摄入并整合到现有知识体系时
* 需要查询 wiki 中的已有知识时
* 需要维护 wiki 的健康状态时

## 目录结构

```
vault/
├── raw/              # 原始资料（不可变，LLM 只读不写）
│   └── assets/       # 图片、附件（原始资料引用的媒体文件归档于此）
└── wiki/             # LLM 生成的 wiki 页面
    ├── entities/     # 实体页面（人物、地点、组织）
    ├── concepts/     # 概念页面（想法、理论、方法论）
    ├── sources/      # 资料摘要
    ├── synthesis/    # 综合分析
    ├── index.md      # 索引
    └── log.md        # 操作日志
```

## 核心操作流程

### 1. 资料摄入 (Ingest)

当有新资料时：

1. **归档原始资料与附件**
   - 将源文件复制到 `raw/`，文件名保持原样
   - **扫描媒体引用**：正则匹配源文件中的所有图片/附件引用，覆盖以下格式：
     - Obsidian wiki-link：`![[image.png]]`、`![[image.png|alt]]`
     - Markdown 相对路径：`![alt](./images/img.png)`、`![alt](../assets/img.png)`
     - Markdown 绝对路径：`![alt](/Users/xxx/img.png)`
     - HTML 标签：`<img src="...">`
     - Base64 内嵌：`![alt](data:image/...;base64,...)` → 提取为文件
   - **定位原始文件**：根据引用类型解析实际文件路径
     - 相对路径 → 以源文件所在目录为基准拼接
     - 绝对路径 → 直接使用
     - Obsidian wiki-link → 在源文件所在目录及其上级搜索
     - Base64 → 提取为 `raw/assets/<hash>.<ext>`
   - **复制附件到 `raw/assets/`**：所有定位到的媒体文件统一复制到 `raw/assets/`，保留原始文件名（同名冲突时加序号）
   - **重写引用路径**：复制后的 md 中，将所有媒体引用统一改写为 `![[filename.ext]]`（Obsidian wiki-link 格式）。Obsidian 会从 `raw/` 向上搜索，自动找到 `raw/assets/` 中的文件，无需指定完整路径
   - **无法处理的引用**：外部 URL（`https://...`）和无法定位的本地路径，保持原样不修改，记录在摘要页的「备注」中
2. **阅读并讨论** - 阅读资料，与用户讨论关键要点
3. **创建资料摘要** - 在 `wiki/sources/` 创建资料摘要页
4. **更新相关页面** - 更新相关的实体/概念页面，添加交叉引用
5. **更新索引** - 更新 `wiki/index.md`
6. **记录日志** - 在 `wiki/log.md` 追加记录

**单个资料可能会更新 10-15 个 wiki 页面。**

### 2. 查询 (Query)

回答问题时：

1. **阅读索引** - 先阅读 `wiki/index.md` 了解 wiki 结构
2. **读取相关页面** - 根据索引找到并读取相关页面
3. **综合答案** - 综合答案，包含页面引用（使用 `[[页面名]]`）
4. **存档有价值的回答** - 好的答案存入 `wiki/synthesis/` 作为新页面

### 3. 维护 (Lint)

定期检查 wiki 健康状态：

- **矛盾** - 页面间的矛盾陈述
- **过时** - 被新资料取代的旧声明
- **孤儿页面** - 没有入站链接的页面
- **缺失页面** - 被提及但没有自己页面的重要概念
- **缺失交叉引用** - 应该存在但没有的链接

## 基础操作

### 列出笔记

获取 vault 中的所有笔记列表。

### 读取笔记

读取指定路径的笔记内容。

### 写入笔记

创建新笔记或更新现有笔记。

### 删除笔记

删除指定的笔记。

### 搜索笔记

在所有笔记中搜索关键词。

### 反向链接

查看哪些笔记链接到了指定笔记。

## 页面格式规范

所有 wiki 页面必须包含 YAML frontmatter：

```markdown
---
title: 页面标题
tags: [tag1, tag2]
created: 2026-04-15
updated: 2026-04-15
source: 资料来源（可选）
---

# 页面标题

内容...
```

## 索引和日志

### index.md

**内容导向**的目录，按类别组织（实体、概念、资料、综合）。每次摄入后更新。回答查询时先读索引。

### log.md

**时间顺序**的记录，追加式：
- 格式：`## [2026-04-15] ingest | 资料标题`
- 记录摄入、查询、维护等操作

## 页面类型规范

### 实体页面 (entities/)

人物、地点、组织等具体实体。

### 概念页面 (concepts/)

想法、理论、方法论等抽象概念。

### 资料摘要 (sources/)

原始资料的摘要页面。

### 综合分析 (synthesis/)

跨资料的综合分析和总结。

## 链接风格

使用 Obsidian 风格的双向链接：`[[页面名]]`

不要包含 `.md` 后缀。

## 命名约定

- 文件名使用 kebab-case：`my-concept.md`
- 页面标题使用正常标题格式
- 所有时间格式：YYYY-MM-DD
