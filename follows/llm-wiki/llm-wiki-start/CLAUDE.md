# Wiki 规范 (Schema)

本文档定义 LLM 如何维护这个 wiki。基于 [llm-wiki.md](../llm-wiki.md) 的核心理念。

## 核心理念

不是每次都从零开始 RAG，而是**维护一个持续进化的 wiki**：

- 原始资料放在 `raw/`（不变）
- LLM 在 `wiki/` 中构建结构化知识
- 知识被编译一次，然后持续更新
- 交叉引用、矛盾标记、综合分析都已存在

## 目录结构

```
wiki-start/
├── CLAUDE.md         # 本文件 - wiki 规范
├── raw/              # 原始资料（不可变，LLM 只读不写）
│   └── assets/       # 图片等附件
└── wiki/             # LLM 生成的 wiki 页面
    ├── entities/     # 实体页面（人物、地点、组织）
    ├── concepts/     # 概念页面（想法、理论、方法论）
    ├── sources/      # 资料摘要
    ├── synthesis/    # 综合分析
    ├── index.md      # 索引
    └── log.md        # 操作日志
```

## 页面格式规范

所有 wiki 页面必须包含 YAML frontmatter：

```markdown
---
title: 页面标题
tags: [tag1, tag2]
created: 2026-04-15
updated: 2026-04-15
source: 资料来源（可选）
---

# 页面标题

内容...
```

## 核心操作流程

### 1. 资料摄入 (Ingest)

当有新资料时：

1. **保存原始资料** - 将原始资料放入 `raw/` 目录，保持原样，不做任何修改
2. **阅读并讨论** - 阅读资料，与用户讨论关键要点
3. **创建资料摘要** - 在 `wiki/sources/` 创建资料摘要页
4. **更新相关页面** - 更新相关的实体/概念页面，添加交叉引用
5. **更新索引** - 更新 `wiki/index.md`
6. **记录日志** - 在 `wiki/log.md` 追加记录

**单个资料可能会更新 10-15 个 wiki 页面。**

### 2. 查询 (Query)

回答问题时：

1. **阅读索引** - 先阅读 `wiki/index.md` 了解 wiki 结构
2. **读取相关页面** - 根据索引找到并读取相关页面
3. **综合答案** - 综合答案，包含页面引用（使用 `[[页面名]]`）
4. **存档有价值的回答** - 好的答案（比较、分析、发现的关联）存入 `wiki/synthesis/` 作为新页面

### 3. 维护 (Lint)

定期检查 wiki 健康状态：

- **矛盾** - 页面间的矛盾陈述
- **过时** - 被新资料取代的旧声明
- **孤儿页面** - 没有入站链接的页面
- **缺失页面** - 被提及但没有自己页面的重要概念
- **缺失交叉引用** - 应该存在但没有的链接
- **数据缺口** - 可以通过网络搜索填补的空白

## 索引和日志

### index.md

**内容导向**的目录，包含：
- 每个页面的链接
- 一句话摘要
- 可选的元数据（日期、来源数）
- 按类别组织（实体、概念、资料、综合）

每次摄入后更新。回答查询时先读索引。

### log.md

**时间顺序**的记录，追加式：
- 格式：`## [2026-04-15] ingest | 资料标题`
- 记录摄入、查询、维护等操作
- 可用简单 unix 工具解析

## 页面类型规范

### 实体页面 (entities/)

命名：人物全名、地点全名等

内容：
- 基本信息
- 与其他实体的关系
- 引用来源

### 概念页面 (concepts/)

命名：概念名称，kebab-case 文件名

内容：
- 定义
- 相关概念
- 应用示例
- 引用来源

### 资料摘要 (sources/)

命名：资料标题，kebab-case 文件名

内容：
- 资料摘要
- 关键要点
- 与现有 wiki 的关联
- 指向原始资料的引用

### 综合分析 (synthesis/)

命名：分析主题，kebab-case 文件名

内容：
- 跨资料的综合
- 比较分析
- 新发现的关联

## 链接风格

使用 Obsidian 风格的双向链接：`[[页面名]]`

不要包含 `.md` 后缀。

## 命名约定

- 文件名使用 kebab-case：`my-concept.md`
- 页面标题使用正常标题格式
- 所有时间格式：YYYY-MM-DD

## 可选工具

规模较大时可考虑：
- [qmd](https://github.com/tobi/qmd) - 本地 markdown 搜索引擎
